# Self-Assessment Matrix (Block 1)

> A quick, no-judgement snapshot of where you are with AI coding *today* — so you can see where you've moved by the end of Day 2.

During the workshop you'll fill this in inside the shared **workshop Google Doc** (Block 1 → Self-Assessment Matrix). This file is a printable copy you can also keep for yourself; nobody will ask you to share it.

---

## How to use this matrix

Pick **one cell per axis** that best matches you *today* — not where you want to be, not where you think you should be. There are no wrong answers; the matrix exists to give the cohort and the facilitator a shared sense of the room.

In the Google Doc version, you'll drop your **initials** into the chosen cells. Here, just circle / highlight / mark them.

## Axis 1 — Tool familiarity

How comfortable are you with agentic AI coding tools (Claude Code, Cursor, Copilot Agent Mode, Open Code, RooCode, Cline, Codex, Antigravity, etc.)?

| Level | What it looks like |
|-------|--------------------|
| **0 — None** | I haven't used any agentic AI coding tool. ChatGPT/Claude in a browser is the most I've done. |
| **1 — Tried it** | I've installed and prodded one tool a few times, but it isn't part of my daily workflow. |
| **2 — Occasional** | I use one tool roughly weekly for small assists — autocompletes, quick refactors, generating tests. |
| **3 — Daily** | One agent is part of my daily flow. I know the slash commands and shortcuts of at least one tool. |
| **4 — Multi-tool** | I regularly use 2+ agentic tools, know their relative strengths, and have opinions about MCP servers, plan mode, or context files. |

## Axis 2 — Autonomy comfort

How much rope do you give the agent on a typical task?

| Level | What it looks like |
|-------|--------------------|
| **0 — Read-only** | I only ask the agent questions. I copy answers manually into my editor; I don't let it write to my files. |
| **1 — Per-edit gating** | I let the agent edit files, but I read and approve every single diff before saving. No commands, no shell. |
| **2 — Approve in batches** | I approve a small batch of edits at once if they look coherent. Shell commands still need explicit approval. |
| **3 — Auto-accept on trusted loops** | For well-bounded tasks (refactors, scaffolding) I let the agent run with auto-accept on, intervening only if I see drift. |
| **4 — Long-horizon delegation** | I routinely hand the agent multi-step tasks (spec-driven, multi-file) and check back after it finishes a coherent slice, not after each edit. |

**My level today:** ____

## Axis 3 — Review habits

How rigorously do you actually review what the agent produced?

| Level | What it looks like |
|-------|--------------------|
| **0 — Run-and-pray** | If the app still runs and tests still pass, I move on. I don't read the diff. |
| **1 — Skim** | I glance at the diff, look for anything obviously weird, then move on. |
| **2 — Spot-check** | I read the changed files end-to-end on important edits; small ones I skim. |
| **3 — Diff every time** | Every agent change gets a `git diff` review before I commit, regardless of size. I commit small. |
| **4 — Review with intent** | I review against my project's conventions (AGENTS.md / styleguide), look for vibe-coding patterns (repetition, drift, outdated APIs), and steer the agent mid-execution rather than after the fact. |
